Implementation

Concept

Create a deck of slides off-screen (outside the ViewBox) and use native SVG animations to bring them on-screen. Ideally slide changes would be triggered by keypresses or mouse clicks. However in SVG1.1 (implemented in all browsers) not all such triggers work natively, so it is currently necessary to use a little javascript (embedded) to achieve the full functionality.

Basic Interactivity

Universally (all browsers and OSes), support mouse clicks to trigger animations. So, using mouse clicks, implement forward and reverse capabilities by accepting clicks on the right half of the screen to advance, and the left half of the screen to go back through the slide deck.

Keyboard Interactivity

SVG1.1 includes the the ideal keyboard interactivity to trigger animations using the begin="accessKey(a)" syntax. Unfortunately few browsers implemented support for this, and now it seems that all those that did originally have now depricated it.

Instead standard events have to be used that rely on javascript listeners. The keydown event is used, since the more obvious onkeypress does not fire for all keys in some browsers (excluding for example ALT, CTRL, SHIFT, ESC). Hence, use onkeydown instead as it fires universally for all keys.

In order to recieve keyboard events, the slides need to have the screen focus. Since SVG elements are not normally focusable, this also requires adding tabindex attributes and managing the focus through the javascript
